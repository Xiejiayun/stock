# 股票量化系统真实需求拆分

## 目标边界

本系统定位为 A 股量化研究与交易决策支持系统，不直接下单。当前 MVP 覆盖身份认证、白名单访问、行情数据获取、基础技术指标、策略信号、风险提示和 Azure Web App 部署。

## 用户与权限

- 只允许白名单用户访问系统。
- 支持 Google 登录，登录成功后仍必须通过邮箱白名单校验。
- 后端必须重新校验 Google ID Token 和白名单，前端白名单判断只用于展示体验。
- 未登录用户不能访问行情、个股、板块、量化信号等业务 API。
- 会话令牌需要签名、过期时间和服务端校验。

## 量化业务能力

- 市场状态：主要指数、涨跌家数、涨跌幅分布、板块热度。
- 个股研究：实时行情、历史 K 线、均线、MACD、RSI、KDJ。
- 策略信号：多指标趋势评分、买入/卖出/观望判断、置信度和解释。
- 风险控制：波动率、最大回撤、趋势状态、仓位建议、止损参考。
- 组合层能力：后续需要加入自选池、回测、持仓约束、交易日志和绩效归因。

## 生产化需求

- 身份配置通过 Azure App Settings 管理，不把密钥写入仓库。
- 数据源失败时必须超时、降级或提示，不能让页面无限加载。
- API 返回需要区分数据可用、部分可用、上游失败和鉴权失败。
- 日志需要记录登录失败、白名单拒绝、数据源失败和策略计算异常。
- 部署流水线需要在构建阶段验证后端 import、前端 build 和健康检查。

## 当前 MVP 交付范围

- FastAPI + React 单体部署到 Azure App Service。
- Google ID Token 登录与白名单校验。
- 签名会话令牌，前端保存到 localStorage 并随 API 请求发送。
- 受保护的行情、搜索、板块、个股和量化 API。
- 基础量化概览接口：系统需求、数据处理需求、策略/风险输出。

## 非目标

- 不提供真实券商下单。
- 不保存用户资金、持仓和交易密码。
- 不承诺行情数据实时性或投资收益。
- 不绕过 Google OAuth、Azure 或数据源访问限制。
