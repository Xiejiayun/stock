# 数据处理需求

## 数据源

- 行情数据来自 AKShare，主要覆盖东方财富公开行情接口。
- 市场数据包含指数行情、A 股实时列表、行业板块排行。
- 个股数据包含实时行情和日线历史 K 线。

## 数据接入要求

- 所有外部数据请求必须设置超时。
- 高频复用数据需要缓存，例如全市场实时行情、指数行情、板块排行。
- 上游失败时优先返回可用旧缓存；没有缓存时返回明确错误或降级状态。
- 数据源异常不能阻塞认证、页面加载和健康检查。

## 标准化字段

- 股票代码统一为 6 位字符串。
- 日期统一为 `YYYY-MM-DD` 字符串。
- 价格、成交量、涨跌幅、估值字段统一转换为 Python 原生数值。
- NaN、空字符串、异常数值需要转换为 `None` 或安全默认值。

## 指标处理

- 计算 MA5、MA10、MA20、MA60。
- 计算 MACD、RSI、KDJ。
- 技术指标需要保留最近 120 个交易日用于前端图表。
- 交易信号需要基于同一份历史数据计算，避免重复请求数据源。

## 风控处理

- 基于历史收盘价计算近 20 日波动率。
- 基于历史净值曲线计算近一年最大回撤。
- 基于均线和动量判断风险等级。
- 输出仓位建议、止损参考和主要风险说明。

## 数据质量输出

- 返回最新数据日期、样本数量、缺失值情况和数据源状态。
- 前端需要展示部分数据不可用状态，而不是无限加载。
- API 应该保留 `partial` 和 `errors` 字段，便于定位上游失败。
